# REQ-002: Complete Linux Syscall Header and Source Coverage

## Status
Accepted

## Statement

picblobs SHALL provide a C header system that exposes Linux system calls as callable C functions, covering all syscalls required by the blob types defined in this specification (alloc-jump, reflective loader, bootstrap stager). Each wrapper function SHALL accept typed arguments, invoke the syscall assembly primitive (REQ-001) with the correct syscall number for the target architecture, and return the raw kernel result. The syscall set SHALL be extensible — adding a new syscall requires only a registry entry and regeneration (see ADR-022).

## Rationale

Blob logic (memory allocation, file operations, socket operations, process control) must interact with the kernel exclusively through syscalls — there is no libc. Providing complete coverage means any blob type, present or future, can use any Linux syscall without requiring new assembly or ABI-level work. The header/source split allows blob authors to include only the headers they need, while the linker (with `-ffunction-sections -fdata-sections --gc-sections`) strips unused syscall wrappers from the final blob.

## Detailed Requirements

### Header Organization

The headers SHALL be organized as one self-contained header per syscall (e.g., `picblobs/sys/mmap.h`), each containing:

1. Per-architecture syscall numbers via `#ifdef` chains (derived from the registry, see ADR-022).
2. OS-specific constants (e.g., `PIC_MAP_ANONYMOUS` varies between default, MIPS, and FreeBSD).
3. The wrapper function as a `static inline` that calls `pic_raw_syscall()`.

Headers are generated by `tools/generate.py` from the canonical registry (`tools/registry.py`).

The following syscall groups are REQUIRED for v1 blob types:

1. **Memory management**: mmap (with mmap2 and old_mmap variants), munmap, mprotect.
2. **File I/O**: read, write, open (with openat fallback on aarch64), close, fstat, lseek, llseek.
3. **Process control**: exit, exit_group.
4. **Socket operations**: socket, connect, accept, bind, listen, setsockopt.
5. **File descriptor operations**: dup2, pipe.

Additional syscalls SHALL be added as new blob types require them. The registry-driven generation ensures consistency across all architectures.

### Syscall Number Tables

Syscall numbers are embedded inline within each wrapper header via `#ifdef` chains selecting the correct number per architecture. The numbers are defined in `tools/registry.py` and generated into the headers by `tools/generate.py`. This replaces the originally-specified separate `syscall_numbers.h` files (see REQ-004 for details on this change).

### Wrapper Function Conventions

Each syscall wrapper function SHALL:

1. Be declared in a header and defined in a corresponding source file (or as a static inline in the header).
2. Accept arguments with types matching the kernel ABI (e.g., `void *addr, size_t length, int prot` for mmap).
3. Pass the correct syscall number and arguments to the assembly primitive.
4. Return the raw kernel return value. Error handling conventions (negative return = error on Linux) SHALL be documented but not transformed — no errno emulation.
5. Be compiled with `-ffunction-sections` so that unused wrappers are eliminated by the linker's `--gc-sections` pass.

### Architecture-Specific Variations

The registry tracks per-architecture traits that affect syscall wrappers:

- **uses_mmap2**: i686, armv5, mipsel32, mipsbe32 use `mmap2` (page-unit offset) instead of `mmap`.
- **uses_old_mmap**: s390x passes mmap arguments via a struct pointer on the stack.
- **openat_only**: aarch64 has no legacy `open` syscall; the `pic_open` wrapper uses `openat` with `AT_FDCWD`.
- **needs_got_reloc / needs_trampoline**: MIPS requires GOT self-relocation (see ADR-020).
- **is_32bit**: Affects type widths in syscall arguments.

Where an architecture uses a variant syscall (e.g., `mmap2`), the generated wrapper transparently handles the translation (e.g., dividing offset by page size).

## Acceptance Criteria

1. Every syscall required by the v1 blob types has a generated wrapper header for every supported architecture.
2. Each wrapper correctly invokes the syscall primitive with the right number and argument mapping.
3. Unused syscall wrappers do not appear in final blob binaries (verified by `--gc-sections`).
4. Architecture-specific variants (mmap2, old_mmap, openat) are handled transparently.
5. Adding a new syscall requires only a registry entry and `generate.py` run.
